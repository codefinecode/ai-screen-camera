# DPIA (Data Protection Impact Assessment) - Skeleton

## Обзор обработки данных

**Система**: Edge AI Intermediate Service  
**Цель**: Сбор анонимных демографических/поведенческих метаданных с камер для аналитики и триггерной смены контента.

## Обрабатываемые данные

### Категории данных
- **Метаданные лиц** (анонимные): возраст, пол, эмоции, время взгляда (dwellTime), время внимания (attentionTime), наличие очков
- **Технические данные**: timestamp, playerUUID, cameraId, contentId
- **Координаты** (опционально): x, y, width, height (позиция лица на экране)

### Что НЕ обрабатывается
- Изображения (imgDataBase64 удаляется до логирования/очередей)
- Эмбеддинги/биометрические шаблоны
- Персональные идентификаторы (нет привязки к конкретным лицам)

## Меры безопасности

1. **Удаление изображений**: imgDataBase64 отбрасывается на этапе валидации, не попадает в логи/очереди/AWS
2. **Анонимизация**: только агрегированные метрики передаются в облако
3. **Шифрование**: TLS для исходящих соединений (в проде)
4. **Хранение секретов**: через .env / AWS Secrets Manager
5. **Логирование**: без персональных данных, ротация логов

## Правовая основа

Обработка анонимных метаданных для аналитики и улучшения пользовательского опыта.

## Риски и меры

| Риск | Вероятность | Влияние | Меры |
|------|-------------|---------|------|
| Утечка изображений | Низкая | Высокое | Строгая валидация, удаление imgDataBase64 до логов |
| Деанонимизация | Низкая | Среднее | Только агрегированные данные, без привязки к лицам |
| Несанкционированный доступ | Средняя | Среднее | TLS, Bearer token, ограничение доступа |

## Контроль и аудит

- Логирование всех операций (без персональных данных)
- Мониторинг через /api/health и /api/metrics
- Регулярный аудит обработки данных
